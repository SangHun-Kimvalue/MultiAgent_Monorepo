> `<...>` 자리를 프로젝트 값으로 채우고 이 안내 블록(> 인용)은 지운다.

## 세션 부트 (새 컨텍스트·압축 직후 필수)

압축 요약은 손실적 — **신뢰 금지**. 착수 전 아래를 **실측**한다:

1. **현재 위치**: `git log --oneline -15` + 진행 SoT 문서를 직접 읽는다 — `<진행 SoT 경로, 예: docs/ROADMAP.md §N>`. 메모리·요약·보고서로 대체하지 않는다.
2. **역할 확인**: 이 세션의 역할(`<설계 / 설계검토 / 구현 / 리뷰 / 커밋 등>`)을 판별하고, 매칭 스킬이 있으면 먼저 호출한다(skills-first). 한 세션이 여러 역할을 겸하지 않는다.
3. **불변 재확인**: `<프로젝트 핵심 불변 — 예: 코드는 verdict/exit code만 분기, 검증된 것만 통과 주장, 커밋 AI attribution 금지>`.
4. **동결 경계**: `<편집 금지 repo/경로>`는 읽기만 한다.

> 왜: 자동 로드되는 건 이 파일 + 메모리 인덱스뿐이다. 설계·로드맵·핸드오프는 직접 Read해야 보인다. 부트 절차가 그 Read를 강제해 "요약이 기억하길 바람"을 "SoT에서 재유도"로 바꾼다.

- 언어/인코딩/커밋 규약: `<...>`

## 메모리 규약

- 메모리에 휘발성 진행상태를 박지 않는다(stale 재발 원인). 메모리 = "어디를 보라" 포인터 + 안정 오리엔테이션만.
- 진행은 항상 위 진행 SoT + `git log`로 실측.
